# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 프로젝트 개요

"자폐 연구의 역사: 빅데이터와 바이오뱅크의 시대" — 한국어 학술 서적 프로젝트.
저자: 안준용 (고려대학교 보건과학대학 바이오시스템의과학부)

HonKit(GitBook 후속) 기반 마크다운 책. 형식은 `../humanbrain/` 프로젝트와 동일.

## 빌드 및 로컬 확인

```bash
npm install          # 의존성 설치 (honkit)
npm run build        # _book/ 디렉토리에 HTML 빌드
npm run serve        # localhost:4000에서 라이브 프리뷰
```

## 프로젝트 구조

```
book.json            # 책 제목, 설정
SUMMARY.md           # 목차 — 파트/챕터 구조 정의
README.md            # 들어가며 (책 소개)
getting-started.md   # 서문
partN/chapterM.md    # 본문 챕터
scripts/             # 유틸리티 스크립트
_book/               # 빌드 출력 (gitignore)
```

SUMMARY.md가 책의 구조를 결정한다. 새 챕터 추가 시 반드시 SUMMARY.md에도 항목 추가.

## 글쓰기 규칙

### 용어

| 금지 | 올바른 표기 |
|------|------------|
| 돌연변이 | 유전변이 / 유전 변이 |
| 드노보 변이 | 신생변이 / 신생유전변이 |
| 자폐증 | 자폐스펙트럼장애 |
| 질병 | 질환 |
| 게놈 / 게놈학 | 유전체 / 유전체학 |
| 질환 취약성 | 사용 금지 — 다른 방식으로 서술 |
| 서명 (signature) | 특징 (예: 전사체 특징) |
| 아이소폼 / 이소폼 | 이소체 |
| 롱리드 / 쇼트리드 | 긴길이 / 짧은길이 (예: 긴길이 시퀀싱) |
| 농축 (enriched) | 빈번하다 / 많이 포함되어 있다 |
| 동정 (identification) | 발견 / 확인 / 밝힌 |

### 문체

1. **굵은 글씨(**) 사용 금지** — 본문에서 마크다운 볼드 쓰지 않음
2. **AI식 과장 금지** — "완전히 뒤집었다", "놀라운 발견", "극적으로", "혁명적인" 등 수사 금지. 평이하고 사실적으로
3. **문단 최소 5문장** — AI가 만드는 2-3문장 문단 금지
4. **인용 형식** — "Author et al. (Year)은" → "Author et al. (Year) 연구는" (조사 직접 붙이지 않고 '연구는' 삽입)
5. **문단 시작 다양화** — 모든 문단을 "XX et al." 인용으로 시작하지 않음. 질문, 개념 설명, 비유 등으로 시작
6. **어려운 용어 풀어 설명** — 전문 용어 처음 등장 시 반드시 풀어 설명하고 비유 첨가
7. **줄임말 규칙** — 이후 본문에서 다시 쓰지 않을 용어는 줄임말(약어) 도입하지 않음
8. **연구자 이름 나열 금지** — "XX대학교의 YY 연구실의 ZZ가 이끈 연구" 식 나열 금지. (Author et al. Year) 인용으로 충분
9. **인명 표기** — Steve Horvath = 호바스 (호르바스 X)
10. **제목 규칙** — 연도/출판 사건 중심 제목 지양. 과학적 내용을 반영하는 제목 사용

### 신경다양성 존중

자폐스펙트럼장애, 조현병 등 신경정신과적 조건을 '취약하다', '고장', '결함', '나쁜 결과'로 프레이밍하지 않는다.

- "취약하다" → "높은 빈도로 나타난다"
- "질환 취약성" → "신경정신과적 조건"
- "고장" → "변이", "분기"
- "뇌를 고장 내는" → "다양한 신경 발달 경로를 만들어내는"

유전학적 사실은 정확히 기술하되, 표현형을 '결함'으로 가치 판단하지 않는다.

### 미출간 논문

심사 중이거나 준비 중인 미출간 논문의 내용은 본문에 쓰지 않는다. 출판된 논문만 인용.
